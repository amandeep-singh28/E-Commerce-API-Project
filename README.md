# E-Commerce API Project

> ⚠️ **Disclaimer:** This project is currently a work in progress and is **not yet completed**.

Welcome to the E-Commerce API! This is a web application structured with a Django and Django REST Framework (DRF) backend, coupled with a vanilla HTML/CSS/JS frontend.

## Project Roles & Contributions

* **Backend Development:** The entire backend architecture, data modelling, APIs, routing, and business logic were built entirely by **Me (the User)**.
* **Frontend Development:** The frontend user interface templates, CSS styling, and integration functionality were generated by **AI**.

## What I Have Done (Backend Implementation)

I designed and implemented the backend data and API endpoints. Here is a brief summary of the specific features I built:

* **Database Design:** Created a robust relational model structure comprising `Category`, `Product`, `Order`, and `OrderItem` models.
* **RESTful endpoints:** Built a complete CRUD RESTful API utilizing ViewSets (`ModelViewSet`) to keep API views scalable and concise.
* **Advanced Serializing:** Customized DRF endpoints. A highlight is the `OrderSerializer` where I overrode the `create()` method. This seamlessly handles nested payload writes, processing an Order and simultaneously creating mapping multiple `OrderItem`s to the database in one single request.
* **Bulk Operations:** Overrode the `get_serializer()` method within `ProductViewSet` so the API can support bulk creation of product items if given a JSON list.
* **Security & Permissions:** Set up endpoint-level security, utilizing built-in permission classes to distinguish Admin flows from Normal User flows. For example, any authenticated user can `GET` products, but only `Admin` users can create or modify them.
* **Stateful Flow:** Seamless JSON Web Token (JWT) integration with `TokenObtainPairView` and `TokenRefreshView`.
* **Query Parameters:** Implemented backend data retrieving refinement allowing Django Filter backends, text string searches (`SearchFilter`), and field ordering (`OrderingFilter`).

## Core Concepts & Technologies Used

### Backend (My Work)
* **Python Frameworks:** Django & Django REST Framework (DRF)
* **Django ORM Relationships:** Usage of `ForeignKey` structures (One-to-Many linking Categories -> Products, Users -> Orders, Orders -> Items) with `CASCADE` options.
* **Authentication:** JWT implementation via `djangorestframework-simplejwt`.
* **DRF Serializers:** Used `ModelSerializer`, implementing `source` extraction and custom object mapping inside `validated_data`.
* **API Views & Routing:** Centralized endpoint handling using `viewsets.ModelViewSet` mapped directly to DRF's `DefaultRouter`.
* **Custom Permissions:** Logic using `IsAuthenticated` and `IsAdminUser` conditional returns.
* **Filtering:** Integrations from `django_filters`.

### Frontend (AI Work)
* Vanilla Stack: HTML5, CSS3, JavaScript.
* Includes separate page environments: standard user site (`user.html`), administrator control (`admin.html`), and login (`login.html`).
* Client-side architecture to securely hold JWT tokens and fetch protected endpoints.
